## Контекст (бизнес-логика раздела Запросы)

Закупщик ведёт переписку с европейскими поставщиками (Hans Mueller & Co., Midwest Parts GmbH и др.).  
Поток работы:

```
ЗПО от менеджера → объединяются в REQ (Excel-файл) → отправляется поставщику
→ поставщик отвечает ценами → закупщик актуализирует позиции → REQ переводится в Обработанные
```

Три статуса запроса на странице `page-quotation`:

| Таб | Смысл |
|---|---|
| **Активные** | ЗПО от менеджеров, ещё не отправленные поставщику |
| **В работе** | REQ-файл отправлен, ведётся переписка с поставщиком |
| **Обработанные** | Запрос завершён, цены актуализированы |

### 6. Кнопка «Завершить» и попап подтверждения

**Бизнес-логика:** Закупщик вручную переводит REQ из статуса «В работе» в «Обработанные». При этом менеджер, создавший запрос, получает уведомление о том, что цены актуализированы.

**Кнопка:**
- Расположение: шапка страницы `page-quotation-detail`, правый верхний угол
- Цвет: зелёная (`#16A34A`)
- `onclick="reqOpenDoneModal('12346','AGCO','Петров И.М.')"`

**Попап (modal-req-done) — двухшаговый:**

Шаг 1 (`#rd-step1`) — подтверждение:
```
Запрос обработан. Перевести в Обработанные?
[Нет]  [Да]
```
- «Нет» → `rdClose()` → закрывает попап, остаётся на странице
- «Да» → `rdConfirm()` → переходит к шагу 2

Шаг 2 (`#rd-step2`) — информационное сообщение:
```
✓ (зелёный круг с галочкой)
Запрос переведён в Обработанные.
Менеджер оповещён.
[Закрыть]
```
- «Закрыть» → `rdClose(); nav('quotation')` → переход в список запросов

**Важно — открытие попапа:**  
Стандартный `openModal(id)` добавляет класс `active` к элементу `modal-{id}`. Но этот попап имеет inline `style="display:none;"`, который перебивает CSS-класс. Поэтому используется прямое управление:
```js
function reqOpenDoneModal(...) {
  var el = document.getElementById('modal-req-done');
  el.style.display = 'flex';
  document.body.style.overflow = 'hidden';
}
function rdClose() {
  document.getElementById('modal-req-done').style.display = 'none';
  document.body.style.overflow = '';
}
```

### 7. Редактирование цен в таблице REQ

**Бизнес-логика:** Поставщик может прислать скорректированные цены — закупщику нужно обновить их в системе. После сохранения менеджер получает уведомление об изменении закупочной стоимости.

**Кнопка «Редактировать»** → `qdToggleEdit()`:
- Ячейки `.qd-amt` (Amount EUR) превращаются в `<input type="text" inputmode="decimal">`
- Тип `text` (не `number`) — чтобы не было стрелок-спиннеров
- В шапке таблицы появляются кнопки «Сохранить» и «Отмена»

**Логика пересчёта** `qdPriceInput(input)`:
- `Total EUR = Amount × Кол-во` пересчитывается в реальном времени
- Если цена изменилась — строка подсвечивается оранжевым фоном
- Бейджи ↑/↓ показывают направление изменения

**Сохранение** `qdSaveEdit()`:
- Собирает массив изменений `[{art, from, to}, ...]`
- Если есть изменения — открывает `modal-price-changes` с перечнем
- Менеджер видит уведомление об обновлении цен

**Атрибуты на строках:**
```html
<tr data-art="GCA5301">
  <td class="qd-qty">2</td>
  <td class="qd-amt" data-orig="138.00">138.00</td>
  <td class="qd-total">276.00</td>
  <td><span class="type-so qd-type-chip" onclick="qdTypeClick(this,event)">SO</span></td>
</tr>
```

### 8. Кликабельные чипы типа заказа (TИП)

**Бизнес-логика:** Тип заказа (SO/STD/AN/BCK/MD/EXPR) определяется автоматически, но иногда требует ручной корректировки — например, если позиция стала бэкордером или нужна экстренная поставка.

**Реализация:**
- Чип `<span class="type-so qd-type-chip" onclick="qdTypeClick(this,event)">SO</span>`
- `qdTypeClick(el, event)` — открывает dropdown `#qd-type-picker` с `position:fixed`
- Позиционирование через `el.getBoundingClientRect()`
- `qdTypeSet(cls, label)` — меняет класс и текст чипа, закрывает dropdown

**Список типов в пикере:**

| Чип | Класс | Описание |
|---|---|---|
| SO | `type-so` | Стандарт John Deere |
| STD | `type-std` | Стандартный заказ |
| AN | `type-an` | John Deere 4 недели |
| BCK | `type-bck` | Backorder — перезаказ |
| MD | `type-md` | Express (Machine Down) |
| EXPR | `type-expr` | Express Manitou/AGCO |

## Технические заметки для разработчика

### Два паттерна открытия модалов (важно!)

В `index.html` существуют ДВА разных способа работы с модалами:

**Паттерн A — через класс `.active`** (стандартный):
```js
// Открыть: openModal('id-без-префикса-modal')
// Закрыть: closeModal('id-без-префикса-modal')
// CSS: .modal-overlay.active { display:flex; }
// HTML: <div id="modal-some-id" class="modal-overlay"> (без inline display:none)
```

**Паттерн B — через inline style** (для модалов с `style="display:none;"`):
```js
// Открыть: el.style.display = 'flex'; document.body.style.overflow = 'hidden';
// Закрыть: el.style.display = 'none'; document.body.style.overflow = '';
// Inline style перебивает CSS-класс, поэтому Паттерн A не работает!
```

`modal-req-done` и `modal-price-changes` используют Паттерн B.
