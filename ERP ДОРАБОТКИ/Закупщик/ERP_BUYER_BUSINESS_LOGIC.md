## Важно: профиль закупщика

Закупщик работает **только с промышленным, судовым и общепромышленным оборудованием.**

**Бренды:** Siemens, ABB, Schneider Electric, Festo, Bosch Rexroth, SICK, Honeywell, Continental, Rockwell, ALFA LAVAL, Beckhoff, TURCK и аналоги.

**Не работает** с сельхозтехникой (John Deere, AGCO, CLAAS, Fendt, Manitou и др.) — это отдельный кабинет (Партнёр СХ, в разработке).

## Кто такой закупщик

Закупщик — связующее звено между менеджерами (партнёрами) и европейскими поставщиками.

**Получает от менеджеров:** заявки на запчасти (ЗПО) с артикулами, количеством и приоритетами.  
**Делает сам:** находит цены, сроки, формирует запросы поставщикам, размещает заказы, контролирует поставку.  
**Передаёт назад:** рассчитанные цены, сроки, статусы по каждой позиции.

---

## Бизнес-процесс: от заявки до склада

```
ЗПО от менеджера
    ↓
Закупщик открывает запрос, заполняет синие колонки (цена, срок, кол-во)
    ↓
Отправляет запрос поставщику (письмо EN/DE/PL)
    ↓
Поставщик отвечает → закупщик актуализирует данные
    ↓
Отправляет в расчёт → КП уходит менеджеру
    ↓
Менеджер согласовывает с клиентом → закупщик размещает заказ
    ↓
Запрашивает проформу → получает → оплачивает → товар в пути
    ↓
Транзит → Таможня → Склад → Контроль приёмки → Готов → Закрыт
```

## 1. Бизнес-логика раздела Запросы

### Три состояния запроса (ЗПО)

| Статус | Смысл | Кто переводит |
|---|---|---|
| **Активные** | ЗПО пришёл от менеджера, ещё не взят в работу | Система (автомат) |
| **В работе** | Закупщик работает: заполняет цены, переписывается | Закупщик → «В работу» |
| **Обработанные** | Цены подтверждены, отправлено в расчёт | Закупщик → «В расчёт» |

### Таблица позиций: зелёные и синие колонки

**Зелёные колонки** — приходят от менеджера, закупщик не редактирует:
- Производитель, Артикул, Наименование ENG/RUS, Кол-во, Доп. инфо

**Синие колонки** — закупщик заполняет сам:
- Вес ед./итого, Кол-во Заказ (может скорректировать), Кол-во Мин (минималка поставщика), Цена EUR, Итого EUR, Срок дн., Фото

### Заполнение синих колонок: два способа

1. **Вручную** — кликнуть ячейку, ввести значение
2. **Вставка из Excel (Ctrl+V)** — скопировать колонки из ответа поставщика → кнопка «Вставить данные» → данные встают с позиции курсора

### Переписка с поставщиком

Закупщик отправляет запрос на английском, немецком или польском — в зависимости от страны поставщика.  
Два шаблона: формальный и неформальный. Контакт выбирается из справочника.

**Правило:** письмо уходит от `aiclaude@agr24.ru`, тема и тело подставляются автоматически по шаблону.

### Чат с менеджером (партнёром)

Закрытый чат внутри запроса — для уточнений (фото шильды, замена артикула, срочность).  
Документы партнёра (ТЗ, чертежи) хранятся во вкладке «Документы» этого чата.

### Сохранение черновика

Каждые 90 секунд — автосохранение в localStorage. Закупщик может откатиться к любому из 10 последних снимков.

### Доп. информация по позиции (кнопка i)

Если менеджер оставил комментарий или фото (шильда оборудования, серийный номер) — кнопка подсвечена.  
Закупщик открывает, смотрит, может прикрепить своё фото для письма поставщику.

## 2. Бизнес-логика раздела Заказы

### Три группы заказов

| Группа | Смысл |
|---|---|
| **К размещению** | Менеджер согласовал КП с клиентом, ждёт размещения |
| **Размещённые** | Заказ отправлен поставщику, товар в пути |
| **Завершённые** | Товар принят, КЛ закрыт |

### Статусная машина (12 шагов)

| Шаг | Статус | Что происходит | Кто переводит |
|---|---|---|---|
| 3 | `awaiting` | Заказ создан, ждёт действий закупщика | Система |
| 3 | `urgent` | Срочный заказ | Менеджер |
| 3 | `proforma_req` | Закупщик отправил запрос проформы поставщику | Закупщик |
| 4 | `proforma_rcv` | Поставщик прислал проформу, подтверждена | Закупщик |
| 5 | `paid` | Оплата отправлена, ответственное лицо зафиксировано | Закупщик |
| 6 | `shipped` | Поставщик подтвердил отгрузку | Закупщик / авто |
| 7 | `transit` | Товар в транзите | Авто |
| 8 | `customs` | Товар на таможне | Авто |
| 9 | `warehouse` | Товар прибыл на склад EU | Авто |
| 10 | `control` | Открыт Контрольный лист, физическая приёмка | Авто (по КЛ) |
| 11 | `ready` | КЛ завершён, товар принят | Авто (по КЛ) |
| 12 | `closed` | Заказ закрыт оператором | Закупщик |

**Правило:** каждый шаг разблокирует следующую кнопку. Перепрыгнуть нельзя.

### Workflow кнопок на странице заказа

```
awaiting/urgent  → [Запросить проформу] → письмо поставщику → proforma_req
proforma_req     → [Проформа получена]  → загрузить PDF → proforma_rcv
proforma_rcv     → [Оплачено]           → выбрать ответственного → paid
paid             → [Заказ размещён]     → shipped
```

### Переписка по заказу

Шаблоны писем для каждого этапа: `proforma` / `urgent` / `followup` / `new`.  
Позиции заказа можно вставить в тело письма одной кнопкой.  
К письму — Excel-вложение с позициями.

## Ключевые JS-функции

### Запросы
```js
openRequest(status)        // открыть запрос (active/work/done)
moveToWork(btn)            // → В работе
actualize()                // → Отправить в расчёт
confirmSendCalc()          // подтвердить отправку
updateReqView(status)      // обновить вид по статусу
```

### Таблица запроса
```js
initRequestDetailTable()   // инициализировать таблицу
calcRow(inp)               // пересчитать итого строки
calcWeight(inp)            // пересчитать вес
pasteIntoTable(text)       // Ctrl+V в позицию курсора
pasteGreenApply(text)      // вставить артикулы (зелёные)
pasteBuyerApply(text)      // вставить цены/сроки (синие)
```

### Сохранения
```js
saveReqSnapshot(auto)      // сохранить снимок
restoreSnapshot(idx)       // восстановить
scheduleAutoSave()         // авто каждые 90 сек
```

### Переписка с поставщиком
```js
scSupplierChange()         // выбрать поставщика
scLang(lang)               // язык (en/de/pl)
scTemplate(n)              // шаблон (1/2)
scSendEmail()              // отправить
scNewThread()              // новая переписка
```

### Чат с партнёром
```js
sendPartnerMsg()           // отправить сообщение
pcTab(tab)                 // вкладка (chat/docs)
pcDownloadDoc(idx)         // скачать документ
```

### Workflow заказа
```js
openOrder(id)              // открыть заказ
toggleOrderEmailBlock()    // показать форму письма
markProformaReceived()     // проформа получена
openOrderPaidModal()       // подтверждение оплаты
confirmOrderPaid()         // подтвердить оплату
placeOrderFinal()          // разместить заказ
sendOrderEmail()           // отправить письмо
orderLogAdd(color, text)   // запись в лог
showOrderToast(msg, type)  // тост-уведомление
```
